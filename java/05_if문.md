## if문

[소스1](src/cond/If1.java)

- 조건이 참인지 확인하고, 그 조건이 참(true)일 경우 코드 블록 실행
- `int age = 20;` 일 때 
    ```text
    if (age >= 18) {"성인입니다."}
    ```
  - `age = 20`으로 조건이 참(true) → `"성인입니다."` 출력
  
  ```text
  if (age < 18) {"미성년자입니다."}
  ```
  - 조건이 거짓(false) → if문 블록이 실행되지 않고 빠져나옴. `"미성년자입니다."` 출력 x

## else문

[소스2](src/cond/If2.java)

- if문에서 만족하는 조건이 없을 시 실행하는 코드

    ```text
    if (조건) {
        //조건이 참일 때 실행
    } else {
        //만족하는 조건이 없을 때 실행
    }
    ```
- else문 사용시 [소스1](src/cond/If1.java) 코드 간략히 가능
  - [소스1](src/cond/If1.java) - 18살 이상일 때 "성인입니다." 출력. 18살 미만일 때 "미성년입니다." 출력
  - [소스2](src/cond/If2.java) - 18살 이상일 때 "성인입니다." 출력, 그렇지 않으면 "미성년입니다." 출력
    
    => 18살이 넘으면 성인이고, 그렇지 않으면 모두 미성년이다

## else if문

#### if문 소스 코드 단점

[소스](src/cond/If3.java)

- 불필요한 조건 검사 - 이미 조건을 만족해도 불필요한 다음 조건을 계속 검사
  - 예) `age = 5` 일 때, 미취학 아동이 이미 출력되지만, 나머지 `if문`을 통해 조건 검사를 모두 실행해야 함
- 코드 효율성 - `age = 8`인 초등학생이 미취학을 체크하는 조건인 `age <= 7`을 통해 이미 8살 이상임을 알 수 있지만, 초등학생임을 나타내는 조건 `age >= 8 && age <= 13`이라는 두가지 조건 모두 수행
  
  => `else if`를 사용해 불필요한 조건 검사를 피하고 코드의 효율성 향상

### else if

[소스](src/cond/If4.java)

- `if문`의 조건이 거짓일 때 다음 조건 검사, 앞선 `if문`이 참이라면 `else if문` 실행 x

  ```text
  if (조건1) {
    //조건1이 참일 시 실행
  } else if (조건2) {
    //조건1이 거짓이고, 조건2가 참일 때 실행
  } else if (조건3) {
    //조건2가 거짓이고, 조건3이 참일 때 실행
  } else {
    //모든 조건이 거짓일 때 실행되는 코드
  } 
  ```
  - `if문`을 하나로 묶은 것. 특정 조건이 만족하면 해당 코드를 실행하고 `if문 전체`를 빠져나옴. 특정 조건을 만족하지 않으면 다음 조건 검사
    - 순서대로 맞는 조건을 찾아보고 맞는 조건이 있으면 딱 1개만 실행
  - `else문 생략 가능`
